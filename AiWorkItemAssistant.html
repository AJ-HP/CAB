How-to Guide

Mastering the workflow from ideation to saved work item.

1

The Entry Point

Access the tool via the left-hand navigation panel in Azure DevOps. Click on "AI Work Item Generator" to open the dedicated workspace.

View Left Panel -> AI Work Item Generator
2

Workflow Selection

Select your Work Item Type (e.g., User Story) and enter the requirements or user pain points. Be as detailed as possible to get the best result.

3

The Review UI

Human in the Loop: The AI generates a draft. Review the content for clinical accuracy and technical detail. Ensure it meets HPG's safety standards before proceeding.

4

The Completion

Once you are satisfied with the generated content, click "Save". The assistant will then persist the work item to the backlog for implementation.

Real Output Case Study

Example: Transforming shorthand into a high-quality "PDF Download" User Story.

Input: Pain Points & Requirements
As a super user or authorised client user, I want to download a finalised referral submission as a PDF from the Case List, so that I can save and store it in my organisation’s systems without relying on screenshots.
Generated Output ● Saved to Backlog

Title: Enable download functionality for finalised referral submissions as PDF

Description

As a super user or authorised client user, I want to download a finalised referral submission as a PDF from the Case List, so that I can save and store it in my organisation’s systems without relying on screenshots.

Scope
  • Provide a downloadable PDF functionality for finalised referral submissions accessible directly from the Case List.
  • Ensure users with appropriate permissions (super users or authorised client users) can access this feature without additional configurations.
Objectives
  • Allow users to easily retrieve referral submissions in a standard document format.
  • Eliminate reliance on manual screenshots, ensuring data accuracy.
Deliverables
  • "Download as PDF" button integrated into the Case List.
  • Dynamic PDF generation containing Referral ID, Timestamp, Client Info, and Status.
  • Browser compatibility and role validation safeguards.

Acceptance Criteria

GIVEN user is logged in and on the Case List,
WHEN they view finalised submissions,
THEN the download option should be clearly visible.
GIVEN an error during PDF generation,
WHEN user attempts download,
THEN a clear error message must be displayed.
